──────────────────────────────────────────────────────────────────────────
PHASE I — DISCOVERY (the basin-quotient state machine)
──────────────────────────────────────────────────────────────────────────

We found a small computer living inside a ring of 16 coupled oscillators.
The ring's stable attractors (twisted states, labeled by winding number q)
form a discrete state alphabet. Perturbations (localized phase kicks) act
as instructions. The transition table T[q][instruction] → q' is measured,
not designed — it is what the ring's mathematics volunteers.

Key findings from the pristine N=16 Kuramoto ring:

  1. It is a genuine state machine. Route-invariance: 72/72 trials.
     Composition: physical two-step path = table². Markov property holds.

  2. It has a clock. Minimum reliable inter-instruction wait matches the
     ring's spectral gap: τ_min = 1/K·(2π/N)² ≈ 6.7 time units.

  3. The instruction set is TYPED and the write is QUANTIZED:
       READ  = winding number (topological, free, robust)
       ERASE = local scalar bump (one-way funnel toward q=0)
       WRITE = shaped phase ramp (+1 per full 2π, else nil)

  4. Edits are LOCAL — for robust states. Detuning one oscillator (a
     "lens") changes the transition table only near the defect. Marginal
     states (|q| ≈ N/4) are nonlocality channels: their globally soft
     basin boundaries carry edits to arbitrarily distant table entries.

  5. Multistability is cheap; ORGANIZATION is the scarce resource. A
     random multistable fabric (130+ attractors) does NOT form a usable
     machine — frozen and scattered edges coexist at every drive level.
     The ring computes because SYMMETRY organizes its landscape.

  6. Exact energetic accounting. Newton-verified saddles (Hessian index
     proof per saddle). Barrier heights: q=0→1 = 2.35, q=1→2 = 1.13,
     q=2→3 < 0.5. Coupling efficiency of 3-site kicks: 0.35.

  7. Instructions act on BOUNDARIES, not measure. Fleet statistics
     (20,000 samples × 2 deltas × 200 rays × 2,000 trials):
       Gaussian basin-size coefficient k: INVARIANT under lens
       Orbit-metric distance (gauge-corrected): INVARIANT under lens
       Escape-head radii (surviving states): INVARIANT under lens
       Plateau probability (alpha ≥ 0.8): NULL (no depression detected)
       Marginal |q|=3 family: DELETED under lens

Instruments shipped: basin_machine.cpp (live demo), bqsm_torus.py
(locality), bqsm_probe.py (Markov batteries), bqsm_energetics.py (exact
landscape), bqsm_batteries.py (random-fabric counterfactual),
ring_furnace.c (batched C engine, 130× throughput from v2→v3 via
edge-walk deriv, DT=0.50, fast_sin, early-exit gate).

──────────────────────────────────────────────────────────────────────────
PHASE II — ARCHITECTURE (the n-ary computer blueprint)
──────────────────────────────────────────────────────────────────────────

The winding-number state space suggests a natural multi-level encoding:

  Ring states: q ∈ {-Q_MAX ... +Q_MAX}, stable for |q| < N/4
  At N=16: 7 states → 2.8 bits/ring
  At N=32: 15 states → 3.9 bits/ring

The energy landscape is monotonic in |q|: V(q) = -N·K·cos(2πq/N).
Deeper basins (lower |q|) = more stable, higher switching cost.
Shallower basins (higher |q|) = faster switching, more volatile.

This creates a natural RADIX CONTROL design space:
  Binary (q∈{0,1}):     1 bit/ring, maximum stability
  Ternary (q∈{0,1,2}):  ~1.6 bits/ring
  Quad (q∈{0,1,2,3}):   2 bits/ring, balanced
  N-ary (full):          2.8 bits/ring, minimum stability

The instruction set maps naturally to logic operations:
  ERASE(q=1) → q=0:  natural NOT gate
  WRITE+1(q=0) → q=1: natural SET
  ERASE chain: q=N→N-1→...→0 (funnel decrement)
  WRITE chain: 0→1→2→...→N (quantized increment)

The 5-tier framework: Register → Instruction → Transition → Execution →
Radix Control. Full blueprint at BQSM_ARCHITECTURE.txt.

Three parallel forks implemented via subagent-driven development:
  Fork A: bqsm_vm.c/h — virtual processor core (1,269 lines)
  Fork B: bqsm_asm.py — assembler, 11 built-in examples (1,426 lines)
  Fork C: bqsm_energy.py — Landauer-bound analysis (650 lines)

Energy advantage decomposition (architecture-level, implementation-agnostic):
  1. Asymmetric costs (READ/ERASE free):        3.0×   (42.8%)
  2. Multi-level encoding (quad=2 bits/symbol): 2.0×   (27.0%)
  3. Physics-guaranteed reset (funnel):         2.2×   (30.2%)
  ─────────────────────────────────────────────────
  Total architectural advantage:               13.0×
  (Plus implementation-dependent E_switch/E_phys ratio — multiplicative.)

──────────────────────────────────────────────────────────────────────────
PHASE III — INFERENCE ENGINE (making old hardware compute)
──────────────────────────────────────────────────────────────────────────

The core insight: BQSM transition tables replace floating-point MACs with
integer table lookups. The table is L1d-resident. No dequantization, no
float conversion, no multiply ALU — just index + load + add.

Key kernels built and benchmarked on the 2007 Harpertown X5472:

  bqsm_inference.c: Pure int8 table-lookup forward pass.
    11.9M inf/s single-core (XOR model, 12 table lookups/inf).
    95M inf/s projected across 8 cores.

  bqsm_matmul.c: Ternary matrix multiply via PRODUCT table.
    1.1 GFLOPS-eq scalar, 1.6 GFLOPS-eq SSE4.1, 5.7 GFLOPS-eq 8-core.
    Memory-bandwidth-bound at 6.5 GB/s on this hardware.

  bqsm_batch.c: SoA-lane batched matmul.
    3.5 GMACs/s raw, 8.2 ternary-eq GMACs/s (7-state encoding).
    6.0× speedup from OpenMP lane parallelism.

  bqsm_simd.c: pshufb-accelerated SIMD matmul — THE BREAKTHROUGH.
    4-state (2-bit) TT fits in one 16-byte XMM register.
    pshufb does 16 parallel table lookups per instruction.
    2.8 GMACs/s single-core, 5.1× over scalar.
    1.1 CPU cycles per MAC — near the 1.0 IPC theoretical ceiling.
    Gemma 2B projected at ~20 tok/s on 8 cores.

  bqsm_convert.py: GGUF → BQSM weight converter.
    Reads GGUF via gguf library, extracts tensor metadata, quantizes to
    7-state, writes .bqsm model file. Gemma 2B: d_model=2304, ffn=9216,
    26 layers. End-to-end pipeline proven.

The SIMD breakthrough validates the founding thesis: on hardware limited
to SSE4.1 (2007 instruction set), BQSM table-lookup matmul achieves
near-peak IPC where traditional float matmul is bottlenecked by
dequantization overhead. The 4-state encoding with pshufb is the sweet
spot — 2 bits per state, 16 MACs per instruction, 16-byte TT in L1d.

──────────────────────────────────────────────────────────────────────────
PRODUCT AMBITION — BQSM Inference Server
──────────────────────────────────────────────────────────────────────────

TARGET: A production-quality, user-ready AI inference server built on the
BQSM matmul kernel. The server presents an OpenAI-compatible REST API
(/v1/chat/completions, /v1/images/generations, /v1/video/generations)
backed by the BQSM inference engine running quantized models from GGUF.

DESIGN PRINCIPLES:
  1. PORTABLE: Pure C + SSE/NEON intrinsics. Single binary. No CUDA,
     no Python runtime needed for inference. Targets x86_64 (SSE4.1+)
     and aarch64 (NEON). Runs on 15-year-old hardware.

  2. EFFICIENT: BQSM pshufb kernel at 1.1 cycles/MAC. 4-state encoding
     with 16-wide SIMD. Streaming weight layout. Model stays in RAM,
     weights streamed through cache.

  3. UNIVERSAL API: OpenAI-compatible endpoints. Drop-in replacement
     for any tool that speaks the OpenAI API. Chat, image generation,
     video generation — all through the same interface.

  4. PROFESSIONAL: Clean single-binary deployment. Systemd service.
     Health checks. Graceful shutdown. Request queuing. Token streaming
     via Server-Sent Events. Proper error handling and logging.

  5. USER-FRIENDLY: `bqsm serve --model gemma-2b.bqsm --port 8080`
     That's it. One command. Model auto-detection from file header.
     Built-in web UI for testing. Documentation embedded in binary.

  6. IMAGE & VIDEO CAPABLE: Integration with Stable Diffusion (via
     stablediffusion.cpp or similar quantized inference engine) and
     video generation models. Same API format, different model backends.
     All models quantized via the BQSM pipeline.
